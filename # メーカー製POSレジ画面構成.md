# メーカー製POSレジ画面構成

## 1 左側の構成

- 上部 → ヘッダと注文明細 → ヘッダには**テーブルNO**、**客数**、**伝票No**があり、**テーブルNo**または**伝票No**をタップしてモーダルから呼び出し
- 下部 → 金額明細
- 注文明細は所定の品数を超えるとページ分割
- 金額明細の位置は固定

## 2.1 右側の構成

- **登録**と**支払**を切り替えて表示
- **登録**  メニューと各種プリセットキーが配置してある ※ 会計方法が**後払い**の場合は、メニューの割り付けはしていない場合が多い
- **支払** テンキー、値割引や各種支払い種別などのプリセットキーが配置してある。

## 2.2 登録画面の配置キーについて

- **個別割勘**  支払う対象のメニューを選んで個別に会計。レシートが個別に発行される。
- **人数割勘**  人数で金額を等分する。余りがある場合は、最終支払者に割り当てられる。
- **最終割勘**  割勘の残りをすべて呼び出して支払うときに使う。
- **直前訂正**  直前におこなったキー操作を取消したい時。※ 例）直前におこなった値引処理を取消す、など。
- **VOID**  会計処理済の伝票に対しておこなう処理。①レシートNoで呼び出して該当の売上処理を取消す ②同様に呼びだして該当の会計処理を変更 ※現金 → クレジット
- **取消**  注文したメニューを取消す。 ※対象のメニューを選択 → **取消**もしくは**数量**ボタンで数量指定してから**取消**
- **中止**  現在操作している会計処理を中止する。
- **領収書**  所定のフォーマットの領収書を発行。
- **両替**  ドロワーを開放する。
- **扱者**  担当者を選択

## 2.3 支払い画面の配置キーについて
- **値引**
- **割引**
- **クレジット**
- **電子決済**
- **商品券（釣あり）**
- **商品券（釣なし）**
- **売掛**  ほぼ使わない
- **取消**  前述と同じ
- **中止**  前述と同じ
※支払い種別の割り当て変更は設定画面で可能
- 現金は**現計**キー
